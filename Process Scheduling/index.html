Process Scheduling

Process scheduling is an important task of the OS that ensures the best utilization of the CPU. Some of the process scheduling algorithms are - First-Come First Served, Shortest Job First, Shortest Remaining Time First, Priority, Round Robin and Multilevel Queue scheduling.

For more details, refer to the Wiki.

Wiki

FCFS

First Come First Served

FCFS executes queued processes and requests by the order of their arrival. It is non-preemptive. It is rarely used in modern operating systems.

Go
SJF

Shortest Job First

SJF selects the waiting process with the smallest execution time to execute next(non-preemptively). It is optimal but practically infeasible scheduling algorithm.

Go
SRTF

Shortest Remaining Time First

SRTF is preemptive version of SJF. The process with the smallest amount of time remaining for completion is executed first.

Go
Priority

Priority Scheduling

This algorithm selects the process with the highest priority to be execute first. Same priority processes are executed on FCFS basis. Commonly used in batch systems.

Go
RR

Round Robin Scheduling

In RR scheduling, all the processes are executed for a fixed time slot (called quantum) in a cyclic way.

Go
MLQS

Multilevel Queue Scheduling

In MLQS there will be a certain number of ready queues, each with an assigned priority and its own scheduling algorithm. For the process in a queue to execute, all the queues of higher priority should be empty.
